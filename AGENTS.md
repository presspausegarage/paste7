## One-paragraph context

paste7 is a Tauri 2 + React + Monaco desktop scratchpad for safe inspection of healthcare interop messages. Two workflows ship today: a **Scratchpad** for paste-and-observe (HL7 v2, HL7 v3, C-CDA, FHIR JSON/XML) and a **DICOM SR** view for file-drop SR-header inspection. A third **OCR ingest** workflow is planned for Phase 6: screenshots of HL7 v2 messages from third-party viewers → OCR → normalization → the existing HL7 v2 walker. PHI is auto-redacted via per-format rule packs. Per-user Windows installer, no admin rights, no network capabilities, in-memory only. Public MIT on GitHub at `presspausegarage/paste7`; v0.1.0 released 2026-07-18. Rescoped 2026-04-30 from a previous radiology informatics toolkit (`health-integrate` → `paste7`); narrowed again 2026-05-01 (DICOM scope to SR-only; pixel-data OCR replaced with the planned HL7-viewer-screenshot OCR workflow).

## Working conventions

- **MIT license, public GitHub**. No vendor branding. No AI-assistant attribution in commits or code comments.
- **Git commits direct** (no AI-attribution footer). Author via `-c user.name="Andy Weston" -c user.email="andy@presspause.dev"`.
- **Save-as, never overwrite.** DICOM redact-and-export writes `<original>.redacted.dcm`; never mutates source files.
- **Never write paste content to disk.** Scratchpad is memory-only by design — enforced by CI lint.
- **CRLF warnings on git operations are harmless** (Windows filesystem).
- **No HL7/FHIR trademark phrasing in product framing.** "Compatible with HL7 v2" or "parses FHIR" is descriptive use (allowed). Branding around the marks is not.

## What not to do

- **Don't rescaffold the Tauri shell.** It is the only thing that survived the 2026-04-30 rescope and is working.
- **Don't reintroduce stripped scope.** PS360 Template Mapper, String Generator, Tool Launcher, Terminal are gone. Don't restore them.
- **Don't add network capabilities.** No SCP listener, no Azure connectivity, no telemetry, no update-server pings beyond Tauri's signed-release fetch. The "no network" threat model is load-bearing for the secure-environment positioning.
- **Don't broaden DICOM scope.** SR objects only (SOP Class `1.2.840.10008.5.1.4.1.1.88.*`), headers only. No CR/CT/MR/US/etc. No pixel-data redaction of any kind. No SR ContentSequence redaction. The 2026-05-01 rescope made this load-bearing.
- **Don't claim DICOM pixel-data redaction.** Removed entirely from scope on 2026-05-01.
- **Don't add image-output workflows.** Phase 6 OCR is image-in / text-out only; we don't re-render images with redacted regions.
- **Don't add vendor or AI-assistant branding** to UI, docs, or commits.
- **Don't revisit "TS core vs Rust core" or "Tauri vs Electron / VS Code extension"** without reading HANDOFF rationale; both were debated and decided.
